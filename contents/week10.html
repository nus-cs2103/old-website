Pre-Tutorial Learning Activities

T9A1. Set up Travis

You can view the status of all projects and get a rough idea of how each member is contributing from this 'Teams Dashboard' page.
Congrats to those who have set up Travis already. Although not compulsory, we highly recommend setting up Travis CI for your project. It will help you to detect bugs early and maintain a good test coverage. Refer to the relevant learning outcome in Address Book Level 4 for more info.

T9A2. Reach V0.3

Similar to V0.2, but follow V0.3 requirements instead. In particular,
  • Do a 'release' on GitHub. The release should include an executable jar file. Refer the 'Making a Release' section of the sample developer guide for more info.
  • Update User Guide and Developer Guide.
    Tips:
    • Each member is required to do a fair share of documentation. We recommend each student to maintain the developer docs and user docs for the components and features she is in charge of.
    • If you use the docs\diagrams\Diagrams.pptx as the source for diagrams, you may want to split it into one pptx file per diagram to minimize merge conflicts

T9A3. Use test case design heuristics

Refer to the relevant learning outcome in Address Book Level 4 and the relevant lecture handout. Try to apply those heuristics in the test cases you write too. Add extra comments in your code (similar to those in StringUtilTest.java) to show how you applied the heuristics.
Note: The java file linked above is a more recent version of the code you cloned in week 7.

T9A4. Use EclEmma to measure test coverage

Refer to the relevant learning outcome in Address Book Level 4.
Install EclEmma Eclipse plugin and use it to measure test coverage in your code.
