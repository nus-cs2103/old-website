Pre-Tutorial Learning Activities

Activity 1 [Individual, 3pts]

Pick task (a) or (b). Ensure at least one member picks each task and no more than one member picks the same concept/rule.

Preferably, try to pick concepts/rules that you think are not applicable to the project.

  1. Relate* three key concepts of Scrum to your project.
    e.g. product backlog, product owner, sprints, sprint backlog, daily scrum, user stories, scrumboard, story points, burndown charts, team velocity, scrum master, etc.
  2. Relate three XP rules to your project.

There is no need to bring any written answers; it is good enough if you can explain your answer verbally.

*“Relate” here means: Is it applicable in your project? If not, why not? Are you using something similar or an alternative in your project? etc.

Activity 2 [Team, 2pts]

Find a platform, a framework, and a library you could have used in the project. One example each for the whole team is enough.

However, all members should know those examples and why each one is considered a platform, a framework, or a library.

Activity 3 - Reach V0.4

Be ready to demo V0.4 using the jar file or the exe file (i.e. not using Eclipse/Visual Studio)

The code should be tagged as V0.4. Remember to dogfood your app.

Activity 4 - Mark & Collate

Mark your code and try the collate tool on it (we use the collate tool to extract out the code written by each of you, for grading purposes).

For more info about the collate tool, see under the ‘V0.5 additional requirements’ [link coming soon] section of the module handbook.

If the collate tool cannot extract your code from the final submission, you are going to lose 10 marks from your final grade.
