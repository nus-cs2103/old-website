Pre-Tutorial Learning Activities

T8A0. Refine your product's user experience

Read the common problems in V0.0 and improve your product design as necessary. Flaws in product design can cost you marks in the final project grading.

T8A1. Define the milestones

  • If you haven't done so yet, create labels and milestones in your team repo as given in our GitHub setup instructions.
  • Record each of the user stories you plan to deliver as an issue in the issue tracker.
    e.g. Title: As a user I can add a deadline
    Description: ... so that I can keep track of my deadlines
  • Assign the type.* and priority.* labels to those issues.
  • Assign the above issues to suitable milestones (in GitHub).
  • Set the deadlines for milestones (in GitHub). Your internal milestones can be set earlier than the deadlines we have set, to give you a buffer.

Note that you can change the milestone plan along the way as necessary. However, if you do not have a working product at the milestone deadline, or the milestone plan doesn't match the project at the milestone, we'll consider that as a 'missed milestone' which will count against the 'project management' component of your final grade.

Use Github Milestones, Issues, PRs, Tags, Releases, and Labels to track your project schedule. You can also use GitHub's new Projects feature to manage the project, but keep it linked to the issue tracker as much as you can.

T8A2. Reach V0.2

As you work towards V0.2, learn from the last week's experience and refine the workflow as necessary.

IMPORTANT: To consider the milestone as 'reached', you should have the following done *before* the tutorial begins:

  • A working product tagged as V0.2 and pushed to the main repo.
  • All tests passing.
  • Milestone V0.2 updated to match the product and closed.
Doing a 'release' on GitHub is optional for V0.2 but compulsory from V0.3.

T8A3. Complete Developer Guide Appendices

Complete the User Guide (as per V0.0) if you haven't done so already.
Complete the appendices of the Developer Guide.
You are not required to (but welcome to) update the other parts of the Developer Guide at this point.

Tutorial 8: V0.2, Defensive Coding

Demo V0.2

  1. Revert your code to the version tagged as V0.2
  2. Demo your V0.2 to the tutor to prove it meets V0.2 milestone requirements.
  3. Run the tests to show they pass.
  4. Show your repo to tutor to confirm that,
    • org and repo names are as expected
    • milestone V0.2 matches the current version
    • milestone V0.2 and its issues are assigned and closed
  5. Show your User Guide and Developer Guide appendices to confirm they have been completed.
Ask your tutor if you are considered as 'reached V0.2'. As mentioned before, missing milestones V0.2 .. V0.5 will be counted against your final project marks.

Plan V0.3

Update V0.3 milestone on GitHub to match your actual plan for the next week.
