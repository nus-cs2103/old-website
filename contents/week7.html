Pre-Tutorial Learning Activities

T6A1. Get started on the project requirements and design

In parallel to other T6 activities you will do with your current Phase B team members, get together with your Phase C team members and start drafting V0.0 deliverables.

Suggested sequence : document product survey --> document user stories --> prioritize use stories --> select user stories to include in your product --> design the product --> write the user guide
Fill up Glossary and NFRs in parallel to the above.

You may use a simple collaboration medium such as Google Docs for the above. You can transfer the content to your GitHub repo at a later point after the content is somewhat stable. Given that you will be converting the document to Markdown later, minimize effort spent on formatting.

Submission: Submission will be in Week 8 (after converting to markdown and pushing to the repo). In tutorial 7, just show your intermediate drafts to the tutor.

You may also set up (this can be done in week 8 too) GitHub organization for your Phase C, as described here.

T6A2. Work in a 2KLoC code base, as a team

Complete the exercise in Revision Control Software: Follow the Feature Branch Workflow [LO-FeatureBranchWorkflow] but do note the following:

  • [Step 1]
    • Organization name to use: CS2103AUG2016-PHASE_B_TEAM_ID. e.g. CS2103AUG2016-W09-B3
    • Repo to be forked to your new organization: Address Book Level 3.
    • Collaborators to add: your Phase B team members.
  • [Step 2]
    • You can either clone the team repo (created above) to your computer or add the team repo as a 'remote' to your existing addressbook-level3 clone (because both repos have the same ancestor).
    • Feature to add: Do the exercise in AddressBook-Level3: Work in a 2KLoC code base [LO-2KLoC], but take note of the following.
      • Discuss with team members and decide which feature each of you will implement (to avoid duplicating the same feature).
      • Name your branch as some-keywords-related-to-the-enhancement e.g. add-edit-command instead of following the usual branch naming convention for tutorial activities.

Submission: This is a team submission. Create a PR from the master branch your team repo to nus-cs2103-AY1617S1/addressbook-level3. PR name is similar to previous tutorial activities, but omit the student name. e.g. [T6A2][T11-B3]
