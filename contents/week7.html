Lecture 7

Part 1 - The View from the Top: Architecture [40 minutes]

[Handout] [Slides]

Overview: The architectural design should break the product into a small number of well-specified components that can be developed in parallel. UML sequence diagrams can be used to figure out the API of those components.

Topics: software architecture, components and APIs, modeling component behavior using seq diagrams, different approaches to design

Resources:

  • [Must watch] A three-minutes video we created for you Game of Tic-Tac-Toe: A worked example of designing architecture and discovering component APIs

Part 2 - Putting up defenses to protect our code [30 minutes]

[Handout] [Slides]

Overview: Proper error handling is a must for any production system. The defensive coding approach can lower the risk of component failure even further.

Topics: assertions, logging, defensive programming

Resources:

  • Refer Address Book (Level 4) Learning Outcomes page for more resources.

Part 3 - How to Avoid a Big Bang: Integrating Software Components [15 minutes]

[Handout] [Slides]

Overview: Your integration approach play an important role in how you implement components. When integrating work done by multiple developers, revision control software can be a big help.

Topics: Integration, Build automation, Continuous Integration, Dependency Management
