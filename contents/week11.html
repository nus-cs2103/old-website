Pre-Tutorial Learning Activities

T10A1. Do a peer code review

  • Refer to the learning outcome [LO-CodeReview]
  • For at least one PR created by you, get a team member to do a code review (on GitHub) before merging the PR.
  • Review at least one PR created by a team member.

T10A2. Reach V0.4

Similar to V0.3, but follow V0.4 requirements instead. In particular,
  • Remember to collate code/tests/docs written by you (see V0.4 Requirements for more info).
    Your tutor will do a brief review of the collated code to give you some feedback about your code quality.
  • Dogfood your product and find ways to improve it. Remember to keep the dogfooding version of the app (i.e. the jar file) in a separate folder as evidence of dogfooding. i.e. Don't use the IDE for dogfooding (Dogfooding is an acceptance test; it should be done in the user environment)
  • Note that there are special requirements for the V0.4 demo which require advance preparations. More details in the 'V0.4 Requirements' link above.

T10A3. Relate Design Patterns to the project

Refer to the learning outcome Apply Design Patterns [LO-DesignPatterns] and the relevant lecture handout. Find which design patterns are used in the AddressBook design. Are they still in your design too?
Find if those patterns can be applied in other places of your design.

T10A4. Distinguish between unit, integration, and system testing.

Refer to the following learning outcomes to find examples of unit/integration/system tests in AddressBook:
  • Write Integration Tests [LO-IntegrationTests]
  • Perform System Testing [LO-SystemTesting]
Find tests in your projects that are
  • Unit tests
  • Integration tests
  • Unit+integration tests
  • System tests
