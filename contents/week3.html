Lecture 3

[50 minutes] Part 1 - Object-Oriented Programming : Basics

[ Handout] [ Slides]

Overview: During the detailed design, which can be complex, it is good to follow a common design paradigm. OO is the paradigm followed most widely in the industry. When using OO, we view a system as interacting objects. Objects represent entities in the real world and encapsulate data and operations related to those entities.

Introduction to OO, Creating an OO solution, Object diagrams, Class diagrams

Resources:

  • [Supplementary handout, *must read if you are new to OO] Object-Oriented Programming with Objective-C, by Apple Inc. Read these sections to understand the OO philosophy.
    • Object-Oriented Programming
    • The Object Model
  • [Book] UML Distilled (3rd ed) by Martin Fowler - A handy guide for most used UML diagrams. Not required for the module, but can be useful for future UML usage.
  • [Tools] Powerpoint is usually good enough as a UML diagramming tool if you are using UML for simple documentation purposes the way we do in this module. If you would like to try something more UML-specific, there are quite a few available.
    e.g. Draw.io (online), yUML.me (online), Gliffy (online), UMLet, Visual Paradigm
